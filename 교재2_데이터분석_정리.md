## 1. NumPy와 행렬

### 1-1. NumPy 개요 (p.2)

**Python의 대표적인 수치연산 라이브러리**

- 파이썬의 기본 기능으로 연산 시 속도가 느리나, NumPy는 **C 언어로 구현된 함수**를 통해 벡터·행렬·텐서 연산을 빠르게 수행
- **LLM, AI 분야에서 필수적**
- 거의 모든 ML/DL 라이브러리는 NumPy 사용

---

### 1-2. NumPy를 이용한 배열(Array) 다루기 (p.3)

**다양한 배열 초기화 함수**

| 함수 | 설명 |
|------|------|
| `np.array([리스트])` | 직접 배열 만들기 |
| `np.zeros((m,n))` | m×n 영(0) 행렬 |
| `np.ones((m,n))` | m×n 일(1) 행렬 |
| `np.arange(start, stop, step)` | 범위 배열 |
| `np.random.randn(m,n)` | 정규분포 난수 배열 |

---

### 1-3. 벡터 연산 (p.4)

**세 가지 핵심 연산**

| 연산 | 설명 |
|------|------|
| **요소별(Elementwise) 연산** | `+`, `-`, `*`, `/` 사칙연산 |
| **내적(Dot Product)** | 벡터의 방향 유사도 측정에 활용 |
| **노름(Norm)** | 벡터 크기 계산 (거리 개념) |

**코드 예시:**

```python
v1, v2 = np.array([1,2,3]), np.array([4,5,6])
print(np.dot(v1, v2))          # 32
print(np.linalg.norm(v1))      # 3.74
```

### 1-4. NumPy 행렬 연산 (p.5)

**2차원 배열(행렬, Matrix)과 주요 연산**

- 데이터, 가중치를 표현하는 가장 일반적인 형식

**주요 연산:**

| 연산 | 문법 |
|------|------|
| 행렬 곱 | `np.matmul(A, B)` 또는 `A @ B` |
| 전치 행렬(Transpose) | `A.T` |

**코드 예시:**

```python
A = np.array([[1,2],[3,4]])
B = np.array([[5,6],[7,8]])
print(A @ B)    # 행렬 곱
print(A.T)      # 전치 행렬
```

### 2-4. 데이터 선택과 필터링 (p.12)

**조건식 기반 데이터 필터링**

- 특정 부분을 선택하거나 조건 만족하는 데이터만 필터링 가능
- 조건식 기반 필터링이 가장 많이 사용됨

**문법:**

```python
df[(df['age'] >= 20) & (df['gender'] == 'male')]   # AND 조건
df[(df['age'] >= 20) | (df['gender'] == 'male')]   # OR 조건
```

| 연산자 | 의미 | 예시 |
|--------|------|------|
| `&` | AND | 20대 이상 + 남성 필터링 |
| `\|` | OR | 20대 이상 또는 남성 필터링 |

## 3. 텍스트 전처리 핵심 기법

**주요 단계 (파이프라인):**

|------|------|------|
| 1 | 정규표현식 | 패턴 매칭 |

---

### 3-2. 정규표현식(Regex) (p.20~21)

**패턴 기반 검색/치환**

- 이메일, 날짜, 숫자 등 추출 가능

**자주 쓰는 패턴:**

| 패턴 | 의미 | 예시 |
|------|------|------|
| `\d+` | 숫자 | 123, 456 |
| `[가-힣]+` | 한글 | 안녕 |
| `[a-zA-Z]+` | 영문 | Hello |
| `\s+` | 공백 문자 | — |
| `\b` | 단어 경계 | — |
| `.` | 모든 문자 (줄바꿈 제외) | — |

**실전 패턴 매칭 예시:**

| 추출 대상 | 패턴 | 매칭 결과 |
|----------|------|----------|
| 이메일 | `[a-zA-Z0-9._+\-]+@[a-z.]+` | support@python.org |
| 날짜 | `\d{4}-\d{2}-\d{2}` | 2024-01-15 |
| 전화번호 | `\d{3}-\d{4}-\d{4}` | 010-1234-5678 |
| 가격 | `[\d,]+원` | 35,000원 |

> 💡 Tip: 최근에는 LLM을 통해 복잡한 정규식도 쉽게 생성 가능!

---

### 3-3. 텍스트 정제(Cleaning) (p.22)

**정제의 목적:**

- HTML 태그, URL, 이모지 제거
- 특수문자 → 공백 치환
- 공백 정리

**코드 예시:**

```python
def clean_text(text):
    text = re.sub(r'<[^>]+>', '', text)          # HTML 태그 제거
    text = re.sub(r'http\S+', '', text)          # URL 제거
    text = re.sub(r'[^가-힣a-zA-Z0-9\s]', ' ', text)  # 특수문자 → 공백
    return text.strip()

print(clean_text("<p>Hello!</p> 방문: http://example.com"))
# Hello 방문
```

### 3-4. 토큰화(Tokenization) (p.23)

**의미 있는 단위로 분리**

- 문장 단위, 단어 단위, 형태소 단위
- LLM에서는 별도의 토크나이저(Tokenizer) 사용

**N-gram:**

- 연속된 n개의 단어 묶음 → 문맥 반영

```python
text = "딥러닝 모델을 학습시키기 위해 데이터를 전처리합니다"
words = text.split()
bigrams = [' '.join(words[i:i+2]) for i in range(len(words)-1)]
print(bigrams[:3])
# ['딥러닝 모델을', '모델을 학습시키기', '학습시키기 위해']
```

### 3-5. 정규화(Normalization) (p.24)

**텍스트를 표준 형태로 변환**

| 기법 | 설명 |
|------|------|
| 대소문자 통일 | 영문 대소문자 일치 |
| 숫자 → NUM 치환 | 숫자를 일괄 대체 |
| 반복 문자 축소 | "와아아아아" → "와아" |
| 불필요한 공백 제거 | 연속 공백 정리 |

**코드 예시:**

```python
text = "와아아아아 대박!!!! 2025년"
text = re.sub(r'\d+', 'NUM', text)           # 숫자 → NUM
text = re.sub(r'(.)\1{2,}', r'\1\1', text)   # 반복 문자 축소
print(text)  # 와아 대박!! NUM년
```

### 3-6. 불용어(Stopwords) 처리 (p.25)

**불필요 단어 제거**

- 의미 없는 조사, 접속사 등 제거
- 예: "이", "있", "것", "그리고"

```python
stopwords = ["이", "있", "것", "그리고"]
text = "매우 중요한 데이터입니다 그리고 분석에 필요합니다"
filtered = [w for w in text.split() if w not in stopwords]
print(filtered)
# ['매우', '중요한', '데이터입니다', '분석에', '필요합니다']
```

### 4-4. Bag of Words (BOW) (p.32~37)

**단어의 출현 횟수를 벡터로 표현하는 알고리즘**

- 문장 내에 어떤 단어가 몇 번 출현하는지를 벡터로 표현

**처리 과정:**

```
원본 문서: "AI 데이터 AI 머신러닝"
    ↓
단어 분리(Tokenization): [AI, 데이터, AI, 머신러닝]
    ↓ (순서 무시!)
단어 가방(순서 X): {AI, AI, 데이터, 머신러닝}  ← 순서 정보 손실!
    ↓
단어 빈도(Counter): {'AI': 2, '데이터': 1, '머신러닝': 1}
    ↓
벡터 표현(Vector Representation):
  전체 단어집: [AI, 데이터, 머신러닝, 딥러닝, ...]
  벡터:        [2,  1,      1,       0,     ...]
```

**특징:**

- AI, 데이터, 머신러닝 외 단어는 **단어 빈도가 0으로 표시**됨
- 결과 벡터는 **희소(sparse)한 벡터** → 대부분의 값이 0 → 메모리 낭비, 계산 비효율

**장단점:**

| 장점 | 단점 |
|------|------|
| 구조가 단순해서 계산이 빠름 | 순서를 무시하기 때문에 복잡한 맥락에서 내용들을 이해하기 어려움 |

### 4-5. TF-IDF (p.38~41)

**TF-IDF = TF × IDF**

- 문서 내 단어 비율(TF)과 단어의 희귀성(IDF)을 결합
- 단어의 **상대적 중요도를 반영한 인덱싱 지수**

#### TF (Term Frequency) — 문서 내 단어 비율

$$
tf(t, d) = 0.5 + \frac{0.5 \times f(t, d)}{\max\{f(w, d) : w \in d\}}
$$

#### IDF (Inverse Document Frequency) — 각 단어의 중요도에 대한 로그 값

$$
idf(t, D) = \log \frac{|D|}{|\{d \in D : t \in d\}|}
$$

- log(전체 문서 수 / 해당 단어가 포함된 문서 수)

#### TF-IDF 결합

$$
tfidf(t, d, D) = tf(t, d) \times idf(t, D)
$$

**핵심 원리:** 희귀한 단어 → 중요성 ↑

**TF-IDF의 역할:**

- 텍스트로 구성된 data를 Bag of Words 방식을 통해 **단어 벡터 위치로 Indexing**
- → **문장 간 비교 가능**

### 4-6. 문서 유사도 (p.42~43)

**두 가지 주요 유사도 측정 방법:**

| 방법 | 설명 | 수식 | 특징 |
|------|------|------|------|
| **Cosine Similarity** | 두 문서 벡터가 얼마나 비슷한 방향인지 | `cos(θ) = A·B / (‖A‖ × ‖B‖)` | 값 범위: -1(반대) ~ 0(직교) ~ 1(동일) |
| **Euclidean Distance** | 두 벡터의 직선 거리 | `d = √Σ(Ai - Bi)²` | 실제 거리 측정, 벡터 크기 영향 받음, 값 범위: 0 ~ ∞ |

> 임베딩 방식과 유사함

**활용:**

- 검색 시스템
- 추천 시스템
- **Retrieval Augmented Generation (RAG)**
