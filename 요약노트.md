# [AI 인증 시험 대비 통합 요약 노트]

## 1. 데이터 분석 (Data Analysis)

> **핵심:** AI 모델링의 기초가 되는 데이터 핸들링 및 시각화 능력 (NumPy, Pandas, Matplotlib)

### A. NumPy (Numerical Python)

* **특징:** 대규모 다차원 배열과 행렬 연산에 최적화됨. 리스트보다 빠르고 메모리 효율적.
* **주요 속성:**
* `ndim`: 차원 수 (예: 1, 2)
* `shape`: 배열의 형태 (예: (3, 4) -> 3행 4열)
* `dtype`: 데이터 타입 (예: float, int)


* **핵심 기능:**
* **브로드캐스팅 (Broadcasting):** 크기가 다른 배열 간의 연산을 가능하게 함 (예: 배열 + 숫자).
* **생성:** `np.array()`, `np.zeros()`(0으로 채움), `np.ones()`(1로 채움), `np.arange()`(범위 지정).
* **인덱싱/슬라이싱:** 데이터의 특정 부분 선택.



### B. Pandas (Python Data Analysis Library)

* **구조:**
* `Series`: 1차원 데이터 (인덱스 + 값).
* `DataFrame`: 2차원 데이터 (행 + 열, 엑셀과 유사).


* **데이터 로드:** `pd.read_csv()`, `pd.read_excel()`.
* **데이터 탐색:**
* `head()`, `tail()`: 상/하위 데이터 확인.
* `info()`: 데이터 타입, 결측치 여부 확인.
* `describe()`: 기술통계량(평균, 표준편차 등) 요약.


* **데이터 선택:**
* `loc[행이름, 열이름]`: 라벨(Label) 기반 선택.
* `iloc[행번호, 열번호]`: 정수 위치(Integer Position) 기반 선택.


* **결측치 처리:**
* `isna()`: 결측치 확인.
* `dropna()`: 결측치 제거.
* `fillna()`: 결측치 대체 (평균, 중앙값 등).


* **데이터 가공:**
* `groupby()`: 그룹별 집계 (sum, mean 등).
* `merge()`: 데이터 병합 (SQL Join과 유사).
* `sort_values()`: 정렬.



### C. 데이터 시각화 & EDA

* **Matplotlib:** 기본 시각화 라이브러리 (`plt.plot`, `plt.bar`, `plt.scatter`).
* **Seaborn:** Matplotlib 기반의 고수준 통계 시각화.
* `heatmap`: 상관관계 분석 시 주로 사용.
* `boxplot`: 데이터 분포 및 이상치(Outlier) 탐지.
* `pairplot`: 변수 간 관계 파악.


* **EDA (탐색적 데이터 분석):** 데이터를 본격적으로 분석하기 전, 그래프나 통계량을 통해 데이터의 특징을 파악하는 과정.

## 2. LLM 기본 (Large Language Models)

> **핵심:** 언어 모델의 발전 과정, Transformer 구조, 토크나이징 및 생성 원리

### A. 모델의 진화

* **RNN (Recurrent Neural Network):** 순차적 데이터 처리에 강하나, 긴 문장에서 앞부분 정보를 잊어버리는 **장기 의존성(Long-term dependency) 문제**와 학습 속도 저하 발생.
* **Transformer (2017, Attention is All You Need):** RNN의 한계를 극복. **Self-Attention** 메커니즘을 통해 문장 내 모든 단어 간의 관계를 병렬로 계산.

### B. Transformer 구조

* **Encoder:** 입력 이해에 특화 (예: BERT - 분류, 개체명 인식).
* **Decoder:** 문장 생성에 특화 (예: GPT - 다음 단어 예측).
* **Encoder-Decoder:** 이해와 생성을 동시에 (예: T5, BART - 번역, 요약).
* **주요 개념:**
* **Self-Attention:** "그것(it)"이 무엇을 가리키는지 문맥을 통해 파악하는 원리.
* **Multi-Head Attention:** 문맥을 여러 관점(Head)에서 동시에 파악.
* **Positional Encoding:** 병렬 처리 시 단어의 순서 정보를 주입.



### C. 토크나이징 (Tokenization)

* **정의:** 텍스트를 모델이 처리할 수 있는 단위(Token)로 쪼개는 과정.
* **Subword Tokenization:** 자주 나오는 단어는 하나로, 희귀한 단어는 쪼개서 처리 (예: "Unfriendly" -> "Un", "friend", "ly").
* 알고리즘: BPE (Byte Pair Encoding), WordPiece.



### D. 생성 제어 (Decoding Strategies)

* **Greedy Search:** 매 단계에서 가장 확률이 높은 단어만 선택 (빠르지만 반복 가능성 있음).
* **Beam Search:** 상위 N개의 가능성을 유지하며 탐색 (더 자연스러움).
* **Temperature:** 창의성 조절 (낮으면 결정론적/보수적, 높으면 무작위성/창의적).
* **Top-k / Top-p (Nucleus):** 확률 분포에서 상위 k개 혹은 누적 확률 p까지의 단어 중에서 샘플링.

## 3. 프롬프트 엔지니어링 (Prompt Engineering)

> **핵심:** LLM의 성능을 최적화하기 위한 입력값 설계 기법

### A. 주요 기법

* **Zero-shot:** 예시 없이 지시문만 제공.
* **Few-shot (In-Context Learning):** 프롬프트 안에 예시(Shot)를 몇 개 제공하여 패턴을 학습시킴.
* **CoT (Chain of Thought):** "단계별로 생각해봐"와 같이 중간 추론 과정을 유도하여 복잡한 문제 해결력 향상.
* **ReAct (Reasoning + Acting):** 생각(추론)과 행동(도구 사용)을 교차하며 작업 수행.

### B. 프롬프트 구성 요소

* **Persona:** "너는 ~ 전문가야"와 같이 역할을 부여하여 답변 품질 향상.
* **Delimiter (구분자):** `"""`, `###` 등을 사용하여 지시문과 텍스트 데이터를 명확히 구분.
* **Format:** 출력 형식을 명시 (예: "JSON 포맷으로 출력해줘").

### C. 문제 해결 전략

* **Hallucination (환각):** 사실이 아닌 것을 사실처럼 말하는 현상.
* 해결: "근거를 대라", "모르면 모른다고 해라" 지시, RAG 활용.


* **Jailbreaking:** 안전 장치를 우회하려는 공격 (프롬프트 방어 기법 필요).

## 4. RAG & Agent (Retrieval-Augmented Generation)

> **핵심:** LLM의 지식 한계(최신성, 폐쇄성)를 외부 데이터 검색으로 극복

### A. RAG 파이프라인 (Indexing -> Retrieval -> Generation)

1. **Indexing (색인):**
* **Chunking:** 긴 문서를 작은 단위(Chunk)로 자름 (Overlap을 주어 문맥 유지).
* **Embedding:** 텍스트를 벡터(숫자 배열)로 변환.
* **Vector DB:** 임베딩된 벡터를 저장 (Pinecone, Chroma, Milvus 등).


2. **Retrieval (검색):**
* 질문을 벡터로 변환 후 Vector DB에서 가장 유사한(Cosine Similarity 등) 청크를 검색.
* **Hybrid Search:** 키워드 검색(BM25) + 의미 검색(Vector) 결합.
* **Re-ranking:** 검색된 결과의 순위를 정밀하게 재조정.


3. **Generation (생성):** 검색된 문서를 프롬프트에 문맥(Context)으로 넣어 LLM이 답변 생성.

### B. Agent (에이전트)

* **개념:** LLM이 스스로 계획(Planning)하고 도구(Tool)를 사용하여 작업을 수행하는 자율 시스템.
* **구성:** Brain (LLM), Memory (이전 기록), Planning (작업 분해), Tools (검색, 계산기, API 등).
* **프레임워크:** LangChain, LangGraph (흐름 제어).
* **Function Calling:** LLM이 외부 함수를 호출할 수 있도록 JSON 형태의 인자를 생성하는 기능.

## 5. Fine Tuning (파인 튜닝)

> **핵심:** 사전 학습된 모델을 특정 도메인이나 작업에 맞게 최적화

### A. 학습 방식

* **Pre-training (사전 학습):** 대량의 데이터로 일반적인 언어 능력 학습.
* **Fine-tuning:** 특정 데이터로 추가 학습.
* **SFT (Supervised Fine-Tuning):** [질문, 정답] 쌍의 데이터로 지도 학습 (Instruction Tuning).
* **RLHF (Reinforcement Learning from Human Feedback):** 인간의 피드백을 보상 모델(Reward Model)로 학습하여 강화학습 (PPO 등 사용). -> Alignment(정렬) 목표.



### B. 효율적 학습 (PEFT: Parameter-Efficient Fine-Tuning)

* **Full Fine-Tuning:** 모든 파라미터를 업데이트 (비용 높음, Catastrophic Forgetting 위험).
* **LoRA (Low-Rank Adaptation):**
* 기존 가중치는 고정(Frozen).
* 학습 가능한 **작은 랭크의 행렬(Rank decomposition matrices)**을 추가하여 학습.
* 메모리 사용량이 적고 학습 속도가 빠름.


* **QLoRA:** LoRA에 **양자화(Quantization, 4bit 등)**를 적용하여 메모리 효율 극대화.

### C. RAG vs Fine-tuning 비교 (시험 단골 문제)

| 구분 | RAG (검색 증강 생성) | Fine-tuning (파인 튜닝) |
| --- | --- | --- |
| **목적** | 최신 정보, 외부 지식 활용 | 말투, 형식, 특정 도메인 지식 내재화 |
| **데이터** | 문서 데이터베이스 (Vector DB) | [질문, 답변] 데이터셋 |
| **장점** | 할루시네이션 감소, 실시간 정보 | 모델의 태도/스타일 교정, 전문성 |
| **비용** | 초기 구축 비용 낮음 | 학습 비용(GPU) 높음 |
