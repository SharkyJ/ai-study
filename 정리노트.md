## 교재 2. 데이터 분석 (Data Analysis)

이 파트는 AI 모델 학습의 전제가 되는 데이터 핸들링과 시각화를 다룹니다. Python 라이브러리의 구체적인 사용법이 핵심입니다.

### 1. NumPy (Numerical Python)

**개요:** 고성능 수치 계산을 위한 라이브러리. 리스트(List)보다 빠르고 메모리를 적게 사용하며, 브로드캐스팅 기능을 지원합니다.

* **배열(Array) 생성 및 속성**
* `np.array([1, 2, 3])`: 리스트를 배열로 변환.
* `ndim`: 차원 확인 (예: 1, 2).
* `shape`: 배열의 크기/형태 (예: (3, 4)는 3행 4열).
* `dtype`: 데이터 타입 (예: `int64`, `float64`).
* `size`: 전체 원소의 개수.


* **배열 초기화 함수**
* `np.zeros((행, 열))`: 0으로 채워진 배열 생성.
* `np.ones((행, 열))`: 1로 채워진 배열 생성.
* `np.full((행, 열), 값)`: 특정 값으로 채워진 배열 생성.
* `np.eye(N)`: N x N 크기의 단위 행렬(대각선이 1) 생성.
* `np.arange(시작, 끝, 간격)`: 범위 내의 수열 생성 (Python의 `range`와 유사).


* **배열 변환 및 연산**
* `reshape(행, 열)`: 데이터 변경 없이 배열의 형태(차원)만 변경.
* **Broadcasting (브로드캐스팅):** 모양(Shape)이 다른 배열 간의 연산을 자동으로 수행 (예: 배열 + 10 -> 모든 원소에 10 더하기).
* `flatten()`: 다차원 배열을 1차원으로 평탄화.


* **인덱싱과 슬라이싱**
* `arr[행, 열]`: 특정 위치의 값 접근.
* `arr[시작:끝]`: 범위 접근.
* **Boolean Indexing:** 조건에 맞는 데이터만 추출 (예: `arr[arr > 5]`).



### 2. Pandas (Python Data Analysis Library)

**개요:** 구조화된 데이터(Tabular Data)를 쉽고 직관적으로 다루기 위한 라이브러리.

* **핵심 데이터 구조**
* **Series:** 1차원 배열, 인덱스(Index)와 값(Value)으로 구성.
* **DataFrame:** 2차원 테이블, 행(Index)과 열(Column)로 구성.


* **데이터 입출력**
* `pd.read_csv('파일.csv')`: CSV 읽기. 옵션: `header=None`(헤더 없음), `index_col=0`(인덱스로 쓸 컬럼 지정).
* `pd.read_excel('파일.xlsx')`: 엑셀 읽기.
* `df.to_csv('파일.csv')`: 저장하기.


* **데이터 확인 및 탐색**
* `head(n)`, `tail(n)`: 상위/하위 n개 행 조회.
* `info()`: 요약 정보 (행/열 개수, 데이터 타입, 결측치 여부, 메모리 사용량).
* `describe()`: 기술통계량 (개수, 평균, 표준편차, 4분위수, 최대/최소) 출력.
* `shape`, `columns`, `index`: 속성 확인.


* **데이터 선택 (Indexing)**
* `df['컬럼명']`: 특정 컬럼 선택 (Series 반환).
* `df[['컬럼1', '컬럼2']]`: 여러 컬럼 선택 (DataFrame 반환).
* **`loc[행이름, 열이름]`**: 라벨(Label) 기반 인덱싱.
* **`iloc[행번호, 열번호]`**: 정수 위치(Integer Position) 기반 인덱싱.


* **데이터 조작 및 전처리**
* **정렬:** `sort_values(by='컬럼', ascending=False)` (내림차순).
* **삭제:** `drop(labels, axis=0/1)` (axis=0: 행 삭제, axis=1: 열 삭제).
* **결측치(NaN) 처리:**
* `isna()`, `isnull()`: 결측치 확인.
* `dropna()`: 결측치가 포함된 행/열 삭제.
* `fillna(값)`: 결측치를 특정 값(평균, 0 등)으로 채움.


* **그룹화:** `df.groupby('기준컬럼').sum()` (그룹별 합계, 평균 등 통계 연산).
* **병합:**
* `pd.concat([df1, df2], axis=0)`: 위아래로 이어 붙이기.
* `pd.merge(df1, df2, on='키', how='inner')`: 특정 키를 기준으로 조인(Join).

## 교재 3. LLM 기본 (Large Language Models)

LLM의 기술적 배경인 Transformer 아키텍처와 모델의 작동 원리를 다룹니다.

### 1. 언어 모델의 진화

* **RNN (Recurrent Neural Network):** 순차적으로 데이터를 처리.
* *문제점:* 입력 길이가 길어지면 앞의 정보를 잊어버리는 **장기 의존성(Long-term dependency)** 문제, **기울기 소실(Vanishing Gradient)** 문제 발생. 순차 처리로 인해 GPU 병렬 연산 효율이 낮음.


* **Transformer (2017):** 구글의 "Attention is All You Need" 논문에서 제안. RNN을 사용하지 않고 **Self-Attention**만으로 시퀀스를 처리. 병렬 연산 가능.

### 2. Transformer 아키텍처 상세

* **Encoder (인코더):** 입력 문장의 문맥을 이해하고 압축. (예: BERT)
* **Decoder (디코더):** 정보를 바탕으로 다음 토큰을 생성. (예: GPT)
* **주요 구성 요소:**
1. **Input Embedding:** 텍스트를 벡터로 변환.
2. **Positional Encoding:** 병렬 처리를 위해 단어의 **위치(순서) 정보**를 벡터에 더해줌.
3. **Self-Attention:** 문장 내의 단어들이 서로 어떤 연관이 있는지 계산. (예: "The animal didn't cross the street because **it** was too tired."에서 **it**이 animal임을 파악).
4. **Multi-Head Attention:** 여러 개의 Attention 메커니즘을 병렬로 수행하여 다양한 문맥적 특징을 포착.
5. **Feed Forward Network:** 비선형 변환 수행.
6. **Residual Connection & Layer Normalization:** 학습 안정화 및 깊은 신경망 학습 가능.



### 3. 주요 LLM 모델 유형

* **Encoder-only Models (Auto-Encoding):**
* 양방향 문맥 이해에 강함.
* 대표 모델: **BERT**, RoBERTa, ALBERT.
* 활용: 텍스트 분류, 개체명 인식(NER), 감성 분석.
* 학습 방식: Masked Language Modeling (빈칸 맞추기).


* **Decoder-only Models (Auto-Regressive):**
* 단방향(왼쪽->오른쪽)으로 다음 단어 예측에 특화.
* 대표 모델: **GPT-3**, GPT-4, LLaMA, Claude.
* 활용: 텍스트 생성, 챗봇.
* 학습 방식: Causal Language Modeling (다음 단어 예측).


* **Encoder-Decoder Models (Sequence-to-Sequence):**
* 입력을 이해하고 새로운 시퀀스 생성.
* 대표 모델: **T5**, BART.
* 활용: 번역, 요약.



### 4. 토크나이징 (Tokenization)

* **개념:** 텍스트를 모델이 이해할 수 있는 최소 단위(Token)로 분할.
* **Subword Tokenization (서브워드 토크나이징):**
* OOV(Out of Vocabulary) 문제를 해결하기 위해 사용.
* 자주 등장하는 단어는 그대로, 희귀한 단어는 더 작은 단위로 쪼갬.
* 예: "smartest" -> "smart" + "est".
* 알고리즘: **BPE (Byte Pair Encoding)**, **WordPiece** (BERT에서 사용).



### 5. 생성 제어 파라미터 (Inference Parameters)

* **Max Tokens:** 생성할 최대 토큰 수 제한.
* **Temperature (온도):** 확률 분포의 평탄화 조절.
* 낮음 (0.1): 확률이 가장 높은 단어 선택 (논리적, 정확, 반복적).
* 높음 (0.8~1.0): 확률이 낮은 단어도 선택 가능 (창의적, 다양성, 할루시네이션 위험).


* **Top-k Sampling:** 확률 상위 k개의 단어 후보 중에서만 샘플링.
* **Top-p (Nucleus) Sampling:** 확률의 누적합이 p(예: 0.9)가 될 때까지의 후보군 중에서 샘플링. (Top-k보다 유동적).
* **Frequency/Presence Penalty:** 이미 등장한 단어의 재등장 확률을 낮춰 반복을 방지.

## 교재 4. 프롬프트 엔지니어링 (Prompt Engineering)

모델의 파라미터를 바꾸지 않고, 입력(프롬프트)을 최적화하여 원하는 결과를 얻는 기술입니다.

### 1. 프롬프트 구성 요소

1. **Instruction (지시문):** 모델이 수행해야 할 작업 (예: "요약해줘", "번역해줘").
2. **Context (문맥):** 작업을 수행하는 데 필요한 배경 정보.
3. **Input Data (입력 데이터):** 처리할 실제 내용.
4. **Output Indicator (출력 지시자):** 결과의 형식 (예: "JSON으로 출력해").

### 2. 주요 프롬프팅 기법

* **Zero-shot:** 예시 없이 지시만 내림. (LLM의 일반화 능력 의존).
* **One-shot / Few-shot (In-Context Learning):** 프롬프트 내에 1개 이상의 예제(입력-출력 쌍)를 제공하여 모델이 패턴을 학습하게 함. 파라미터 업데이트 없음.
* **CoT (Chain of Thought):**
* 복잡한 추론 문제를 풀 때 **"단계별로 생각해봐(Let's think step by step)"**라고 유도.
* 중간 추론 과정을 생성하게 하여 정답률을 비약적으로 높임.
* Zero-shot CoT: 예시 없이 "단계별로 생각해봐" 문구만 추가.
* Few-shot CoT: 추론 과정이 포함된 예시를 제공.


* **Self-Consistency (자기 일관성):**
* 동일한 질문에 대해 CoT를 통해 여러 개의 답변을 생성.
* 가장 많이 나온 답변(다수결)을 최종 정답으로 선택.


* **ReAct (Reasoning + Acting):**
* LLM이 추론(Thought)하고, 필요한 행동(Action, 예: 검색)을 결정하고, 결과(Observation)를 받아 다시 추론하는 과정을 반복.
* 에이전트(Agent) 시스템의 기초.



### 3. 고급 전략 및 팁

* **Persona Prompting:** "너는 20년 경력의 시니어 개발자야"와 같이 페르소나(역할) 부여.
* **Delimiter (구분자) 사용:** `"""`, `###`, `---` 등을 사용하여 지시문과 텍스트를 명확히 구분.
* **구조화된 출력:** JSON, XML, Markdown 표 등으로 출력 형식을 명시.

### 4. LLM의 한계와 대응 (Safety)

* **Hallucination (환각):** 사실이 아닌 정보를 사실처럼 생성.
* 대응: "모르면 모른다고 답해", "출처를 표기해", RAG 사용.


* **Prompt Injection / Jailbreaking:** 모델의 안전 필터를 우회하여 유해한 답변을 유도하는 공격.
* 대응: 입력 길이 제한, 키워드 필터링, 샌드위치 방어(지시문을 앞뒤로 배치).

## 교재 5. RAG & Agent (검색 증강 생성 및 에이전트)

LLM의 단점(최신 정보 부재, 할루시네이션, 비공개 데이터 접근 불가)을 해결하는 핵심 기술입니다.

### 1. RAG (Retrieval-Augmented Generation) 개념

* **정의:** LLM이 답변을 생성하기 전에, 신뢰할 수 있는 외부 지식 베이스(Knowledge Base)에서 관련 정보를 **검색(Retrieval)**하고, 이를 프롬프트에 **증강(Augmented)**하여 답변을 **생성(Generation)**하는 기술.

### 2. RAG 파이프라인 상세

1. **Data Ingestion & Indexing (데이터 적재 및 색인)**
* **Document Loading:** PDF, HTML, Word 등 문서를 텍스트로 변환.
* **Text Splitting (Chunking):** 문맥을 고려하여 문서를 작은 단위(Chunk)로 자름.
* 중요: 청크 사이에 **Overlap(중첩)**을 두어 문맥 단절 방지.


* **Embedding:** 텍스트 청크를 숫자 벡터(Vector)로 변환 (OpenAI embedding, HuggingFace model 등 사용).
* **Vector Store:** 임베딩된 벡터를 저장하는 데이터베이스 (Pinecone, ChromaDB, Milvus, Weaviate).


2. **Retrieval (검색)**
* 사용자 질문을 벡터로 변환.
* Vector DB에서 질문 벡터와 가장 유사한(거리가 가까운) 문서 청크를 검색.
* **유사도 측정:** Cosine Similarity(코사인 유사도 - 방향), Euclidean Distance(유클리드 거리).
* **Hybrid Search:** 키워드 검색(BM25/Sparse) + 시맨틱 검색(Vector/Dense)을 결합하여 정확도 향상.
* **Re-ranking:** 검색된 상위 n개의 후보를 정밀한 모델(Cross-Encoder)로 다시 순위 매김.


3. **Generation (생성)**
* 검색된 청크들을 프롬프트의 'Context' 부분에 삽입.
* LLM에게 "Context를 바탕으로 질문에 답해줘"라고 요청.



### 3. Agent (에이전트)

* **개념:** LLM을 단순한 응답 생성기가 아닌, **추론 엔진(Reasoning Engine)**으로 사용하여 목표 달성을 위해 스스로 계획하고 도구를 사용하는 시스템.
* **구성 요소:**
* **Brain:** LLM (계획 및 결정).
* **Memory:** 과거 상호작용 기억 (Short-term / Long-term).
* **Tools:** 외부 세계와 상호작용 (웹 검색, 계산기, 코드 실행, API 호출).
* **Planning:** 복잡한 작업을 하위 작업으로 분해 (Decomposition), 성찰 (Reflection).


* **Function Calling (Tool Calling):** 최신 LLM이 함수(도구)를 실행할 수 있도록, 함수 이름과 인자(Arguments)를 JSON 형태로 생성해주는 기능.

## 교재 6. Fine Tuning (파인 튜닝)

사전 학습(Pre-training)된 모델을 특정 목적 데이터로 추가 학습시켜 최적화하는 과정입니다.

### 1. 학습 단계 및 종류

* **Pre-training (사전 학습):** 인터넷의 방대한 데이터로 일반적인 언어 능력을 비지도 학습. (비용 막대함).
* **Transfer Learning (전이 학습):** 사전 학습된 지식을 기반으로 새로운 작업에 적응.
* **Fine-Tuning:**
* **SFT (Supervised Fine-Tuning):** [Instruction, Input, Output] 쌍으로 구성된 데이터셋을 사용하여 지도 학습. 모델이 지시를 따르는 능력(Instruction Following) 향상.
* **Alignment (정렬):** 모델을 인간의 가치관(도움됨, 해롭지 않음, 정직함)에 맞추는 과정.



### 2. RLHF (Reinforcement Learning from Human Feedback)

* SFT 이후 모델을 더 정교하게 튜닝하는 기법. (ChatGPT 등에서 사용).
* **프로세스:**
1. **SFT 모델 학습:** 양질의 데이터로 초기 튜닝.
2. **Reward Model (보상 모델) 학습:** 모델이 생성한 여러 답변에 대해 사람이 순위(선호도)를 매기고, 이를 예측하는 보상 모델 학습.
3. **PPO (Proximal Policy Optimization):** 보상 모델의 점수를 최대화하는 방향으로 강화학습(RL) 수행.



### 3. PEFT (Parameter-Efficient Fine-Tuning)

* **배경:** LLM의 모든 파라미터(수십억 개)를 튜닝하는 **Full Fine-tuning**은 비용이 너무 크고 **Catastrophic Forgetting(기존 지식 망각)** 문제가 발생.
* **개념:** 모델의 대부분 파라미터를 고정(Freeze)하고, **아주 일부의 파라미터만 추가/학습**하여 효율을 높임.
* **대표 기술: LoRA (Low-Rank Adaptation)**
* 기존 가중치 행렬  옆에 학습 가능한 저랭크 행렬 쌍 를 붙여서 학습 ().
* 학습 파라미터 수를 1% 미만으로 줄임.
* 메모리 절약, 학습 속도 향상, 추론 시 오버헤드 없음(Merge 가능).


* **QLoRA:** LoRA에 **양자화(Quantization, 4bit)** 기술을 결합하여 GPU 메모리 사용량을 극적으로 줄임. (소비자용 GPU에서도 학습 가능).

### 4. RAG vs Fine-tuning 비교 (중요)

시험에서 두 기술의 차이와 선택 기준을 묻는 경우가 많습니다.

| 특징 | RAG (검색 증강 생성) | Fine-tuning (파인 튜닝) |
| --- | --- | --- |
| **목적** | **외부 지식 연결**, 최신 정보 반영, 환각 감소 | **특정 도메인 지식 주입**, 말투/형식 학습, 성능 최적화 |
| **데이터** | 문서 (PDF, txt 등) -> 벡터 DB | 질문-답변 쌍 (Instruction Dataset) |
| **장점** | 실시간 데이터 반영 가능, 답변 근거 확인 가능 | 프롬프트 길이를 줄일 수 있음, 복잡한 지시 수행 능력 향상 |
| **단점** | 검색 정확도에 의존, 시스템 복잡도 증가 | 최신 정보 반영 어려움(재학습 필요), 할루시네이션 가능성 잔존 |
| **적합 사례** | 사내 규정 검색 챗봇, 뉴스 요약 | 의료/법률 진단서 작성, 특정 캐릭터 챗봇, 코드 생성 |
